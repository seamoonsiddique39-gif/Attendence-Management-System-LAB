Attendance Management System
1. Introduction
1.1 Purpose
The purpose of this Attendance Management System is to automate and simplify the process of recording and managing student attendance in educational institutions. The system reduces manual effort, improves accuracy, and provides easy access to attendance records.
1.2 Scope
This system will:
•	Allow teachers to mark student attendance digitally 
•	Allow students to view their attendance records 
•	Allow admin to manage users, classes, and system data 
•	Generate attendance reports for analysis 
The system is designed for schools, colleges, and universities.
1.3 Definitions, Acronyms, and Abbreviations
•	Admin: System manager with full access 
•	Teacher: User who marks attendance 
•	Student: User who views attendance 
•	Attendance: Record of student presence or absence 
1.4 Overview
This document describes the functional and non-functional requirements of the system and provides a clear understanding of system behavior.
2. Overall Description
2.1 Product Perspective
The system is a web-based application that replaces manual attendance registers with a digital solution.

2.2 System Users
•	Admin 
•	Teacher 
•	Student 
2.3 Assumptions
•	Each user has a unique ID 
•	Internet access is available 
•	Teachers are pre-registered by admin 
2.4 Constraints
•	Requires internet connection 
•	Requires database for storage 
•	Users must authenticate before access 
2.5 Dependencies
•	Web browser 
•	Database system (MySQL/MongoDB) 
•	Server environment 
3. Functional Requirements
FR1: User Authentication
The system shall allow Admin, Teacher, and Student to login using valid credentials.
FR2: Attendance Marking
The system shall allow teachers to mark attendance of students for each class session.
FR3: Attendance Storage
The system shall store all attendance records securely in a database.
FR4: Attendance Viewing
The system shall allow students to view their attendance history.
FR5: User Management
The system shall allow admin to add, update, and delete users and classes.
FR6: Report Generation
The system shall generate attendance reports for students and classes.
4. Non-Functional Requirements
NFR1: Performance
The system shall respond within 2 seconds for all requests.
NFR2: Security
The system shall use authentication and role-based access control.
NFR3: Reliability
The system shall ensure 99% uptime.
NFR4: Usability
The system shall have a simple and user-friendly interface.
NFR5: Scalability
The system shall support multiple classes and large numbers of users.
